## CT-103 — Playbooks / DNS / Mail

### Datos del contenedor

| Campo | Valor |
|-------|-------|
| CT ID | 103 |
| Hostname | playbooks-dns |
| Imagen | debian-11-standard_11.7-1_amd64.tar.zst |
| Memoria | 512 MB |
| Disco | 8 GB (local-lvm) |
| Cores | 1 |
| Privilegiado | No |
| Bridge | vmbr1 |
| IP | 10.1.1.5/24 (estática) |
| Gateway | 10.1.1.1 (OpenWRT) |

### Comando de creación

```bash
pct create 103 /var/lib/vz/template/cache/debian-11-standard_11.7-1_amd64.tar.zst \
  --hostname playbooks-dns \
  --memory 512 \
  --cores 1 \
  --rootfs local-lvm:8 \
  --net0 name=eth0,bridge=vmbr1,ip=10.1.1.5/24,gw=10.1.1.1 \
  --ostype debian \
  --unprivileged 1 \
  --password
```

## Servicios instalados

### 1. Postfix + Dovecot — Servidor de correo interno

#### Instalación

```bash
apt update && apt upgrade -y
apt install -y bind9 bind9utils dnsutils curl wget git mailutils postfix
apt install -y dovecot-core dovecot-imapd dovecot-pop3d
```

#### Configuración Postfix

```bash
dpkg-reconfigure postfix
# Opciones seleccionadas:
# - Tipo: Internet Site
# - System mail name: playbooks-dns.lan
```

#### Configuración Dovecot

```bash
# Formato buzón maildir
sed -i 's|#mail_location =.*|mail_location = maildir:~/Maildir|' /etc/dovecot/conf.d/10-mail.conf

# Autenticación
sed -i 's|#disable_plaintext_auth = yes|disable_plaintext_auth = no|' /etc/dovecot/conf.d/10-auth.conf
sed -i 's|auth_mechanisms = plain|auth_mechanisms = plain login|' /etc/dovecot/conf.d/10-auth.conf
```

```bash
# Integración Postfix + Dovecot
cat >> /etc/postfix/main.cf << 'EOF'

home_mailbox = Maildir/
smtpd_sasl_type = dovecot
smtpd_sasl_path = private/auth
smtpd_sasl_auth_enable = yes
EOF
```

#### Usuario de correo SOC

```bash
useradd -m -s /bin/bash socadmin
passwd socadmin
mkdir -p /home/socadmin/Maildir/{cur,new,tmp}
chown -R socadmin:socadmin /home/socadmin/Maildir
```

#### Puertos activos

| Puerto | Servicio | Protocolo |
|--------|----------|-----------|
| 25 | Postfix | SMTP |
| 110 | Dovecot | POP3 |
| 143 | Dovecot | IMAP |
| 993 | Dovecot | IMAPS (SSL) |
| 995 | Dovecot | POP3S (SSL) |

#### Verificación

```bash
ss -tlnp | grep -E "25|143|110|993|995"
# Envío de prueba
echo "Test servidor de correo SOC" | mail -s "SOC Test" socadmin@localhost
# Verificar recepción
ls /home/socadmin/Maildir/new/
```

### 2. Bind9 — DNS interno

#### Zona directa `/etc/bind/zones/db.soc.local`

```
$TTL 604800
@   IN  SOA     playbooks-dns.soc.local. admin.soc.local. (
                2026031601  ; Serial
                604800      ; Refresh
                86400       ; Retry
                2419200     ; Expire
                604800 )    ; Negative Cache TTL

@               IN  NS      playbooks-dns.soc.local.

playbooks-dns   IN  A   10.1.1.5
openwrt-fw      IN  A   10.1.1.1
grafana         IN  A   10.1.1.20
prometheus      IN  A   10.1.1.21
soar            IN  A   10.1.1.30
windows-ad      IN  A   10.1.1.100
backup          IN  A   10.1.1.50
```

#### Zona inversa `/etc/bind/zones/db.10.1.1`

```
$TTL 604800
@   IN  SOA     playbooks-dns.soc.local. admin.soc.local. (
                2026031601  ; Serial
                604800      ; Refresh
                86400       ; Retry
                2419200     ; Expire
                604800 )    ; Negative Cache TTL

@       IN  NS      playbooks-dns.soc.local.

1       IN  PTR     openwrt-fw.soc.local.
5       IN  PTR     playbooks-dns.soc.local.
20      IN  PTR     grafana.soc.local.
21      IN  PTR     prometheus.soc.local.
30      IN  PTR     soar.soc.local.
50      IN  PTR     backup.soc.local.
100     IN  PTR     windows-ad.soc.local.
```

#### `/etc/bind/named.conf.local`

```
zone "soc.local" {
    type master;
    file "/etc/bind/zones/db.soc.local";
};

zone "1.1.10.in-addr.arpa" {
    type master;
    file "/etc/bind/zones/db.10.1.1";
};
```

#### Verificación

```bash
named-checkconf
named-checkzone soc.local /etc/bind/zones/db.soc.local
named-checkzone 1.1.10.in-addr.arpa /etc/bind/zones/db.10.1.1
systemctl restart bind9

# Pruebas
dig @10.1.1.5 grafana.soc.local       # Resolución directa
dig @10.1.1.5 -x 10.1.1.20            # Resolución inversa
dig @10.1.1.5 google.com              # Resolución externa
```
