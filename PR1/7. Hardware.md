## Infraestructura del Proyecto

La infraestructura del proyecto está compuesta por dos servidores físicos principales, encargados de la virtualización de servicios y del almacenamiento centralizado de datos. Esta separación permite mejorar el rendimiento, la escalabilidad y la seguridad del entorno.

---

## 1. Servidor de virtualización (Proxmox VE)

> Nodo principal encargado de la gestión y ejecución de máquinas virtuales y contenedores LXC

**Equipo:** HP Genérico  
**Sistema Operativo:** Proxmox VE  

Este servidor constituye el núcleo de la infraestructura, ya que aloja todos los servicios virtualizados del laboratorio, incluyendo sistemas de seguridad, monitorización, red y automatización.

### Especificaciones técnicas

| Componente        | Detalle                                      |
|------------------|----------------------------------------------|
| **CPU**          | Intel Core i5-3470 @ 3.20 GHz (4 núcleos / 8 hilos) |
| **RAM**          | 16 GB                                        |
| **Almacenamiento** | 200 GB SSD/HDD                             |
| **Red**          | Adaptador de red Gigabit Ethernet           |

---

## 2. Servidor de almacenamiento (Ubuntu Server)

> Nodo dedicado a almacenamiento centralizado y gestión de copias de seguridad

**Equipo:** Dell OptiPlex 5260 AIO  
**Sistema Operativo:** Debian Server  

Este servidor se utiliza como sistema de almacenamiento centralizado para backups y datos del sistema, proporcionando redundancia y soporte a la infraestructura principal.

### Especificaciones técnicas

| Componente        | Detalle                                      |
|------------------|----------------------------------------------|
| **CPU**          | Intel Core i5-8500 @ 3.00 GHz               |
| **RAM**          | 7.6 GB                                       |
| **Almacenamiento** | 238.43 GB SSD/HDD                          |
| **Red**          | Adaptador de red Gigabit Ethernet           |

---

## 3. Arquitectura general

La combinación de ambos servidores permite separar claramente:

- **Cómputo y virtualización** (Proxmox VE)
- **Almacenamiento y backups** (Debian Server)

Esta arquitectura mejora la disponibilidad del sistema, facilita la gestión de recursos y permite escalar el entorno de forma más eficiente.
