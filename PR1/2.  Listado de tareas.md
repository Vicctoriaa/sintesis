# Implementación de la Infraestructura

Este documento describe el proceso de despliegue de la infraestructura representada en el diagrama de red.  
La arquitectura está diseñada para un entorno de **laboratorio de ciberseguridad y monitorización**, utilizando virtualización, segmentación de red, IDS/IPS, monitorización y automatización de respuesta.

---

# 1. Preparación de la infraestructura física

## 1.1 Conexión a Internet
1. Conectar la red del laboratorio al **ISP**.
2. Configurar el **router del taller** para proporcionar acceso a internet.
3. Verificar conectividad básica mediante:
   - `ping`
   - `traceroute`
   - `nslookup`

---
---

# 2. Configuración del servidor de virtualización

## 2.1 Instalación de Proxmox

Instalar **Proxmox VE** en el servidor principal.

Pasos:

1. Descargar la ISO oficial.
2. Instalar el sistema en el servidor físico.
3. Configurar:
   - IP estática
   - Gateway
   - DNS
4. Acceder al panel web:
https://IP_DEL_SERVIDOR:8006


---

## 2.2 Configuración de redes virtuales

Crear bridges de red en Proxmox:

| Bridge | Uso |
|------|------|
| vmbr0 | acceso a red externa |
| vmbr1 | red interna laboratorio |
| vmbr2 | red servicios |

Asignar estos bridges a las máquinas virtuales según su función.

---

# 3. Implementación del Firewall

## 3.1 Despliegue de OpenWRT

Crear una máquina virtual para el firewall.

Configuración:

- 2 interfaces de red
  - WAN
  - LAN

Funciones:

- NAT
- Firewall
- Segmentación de red
- Control de tráfico

Reglas recomendadas:

- Bloquear tráfico no autorizado
- Permitir tráfico interno necesario
- Registrar intentos de acceso

---

# 4. Despliegue de sistemas de seguridad

## 4.1 Implementación del IDS

Crear una máquina virtual **CT-101** para el sistema IDS.

Instalar:

- Suricata **o**
- Snort

Configuraciones:

- Activar reglas de detección
- Monitorizar tráfico de red
- Enviar logs a sistema de monitorización

---

## 4.2 Implementación del Honeypot

Crear la máquina virtual:
VM-201 Honeypot


Objetivo:

- Simular servicios vulnerables
- Detectar ataques
- Registrar comportamiento de intrusos

---

# 5. Sistema de monitorización

## 5.1 Instalación de Prometheus

Desplegar contenedor o máquina virtual **CT-102**.

Funciones:

- Recolección de métricas
- Monitorización de sistemas

Configurar:

- Exporters
- Targets de monitorización

---

## 5.2 Instalación de Grafana

Instalar Grafana en el mismo servidor o en uno dedicado.

Funciones:

- Visualización de métricas
- Creación de dashboards

Integrar con:

- Prometheus
- Logs del IDS

---

# 6. Automatización y respuesta (SOAR)

## 6.1 Servidor de Playbooks y DNS

Desplegar contenedor **CT-103**.

Funciones:

- Ejecutar playbooks automatizados
- Resolver nombres internos
- Automatizar respuestas a incidentes

Ejemplos:

- Bloqueo automático de IP maliciosas
- Alertas automáticas

---

## 6.2 Servidor SOAR Web

Crear contenedor **CT-104**.

Tecnologías:

- PHP
- Base de datos MariaDB/MySQL
- SMTP para alertas

Funciones:

- Panel web de gestión
- Visualización de alertas
- Control de playbooks

---

# Checklist
Para poder desarrollar el proyecto será necesario realizar una serie de tareas organizadas en diferentes fases.

Primero se realizará la instalación del entorno base, que incluye la instalación de **Proxmox** y la configuración inicial del servidor. Posteriormente se crearán los contenedores y máquinas virtuales necesarios para los distintos servicios.

Las principales tareas serán:

- [ ] Instalación y configuración de Proxmox
- [ ] Creación de contenedores y máquinas virtuales
- [ ] Configuración del firewall y DHCP
- [ ] Configuración DNS
- [ ] Instalación del IDS (Snort / Suricata)
- [ ] Configuración de Grafana y Prometheus
- [ ] Desarrollo del SOAR Web
- [ ] Implementación de playbooks
- [ ] Configuración de Active Directory
- [ ] Implementación del honeypot
- [ ] Realización de pruebas (pentesting)
- [ ] Documentación final
