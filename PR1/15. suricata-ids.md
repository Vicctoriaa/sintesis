## Configuración de interfaces (af-packet)

Sección af-packet en `/etc/suricata/suricata.yaml` (línea 588):

```yaml
af-packet:
  - interface: eth0
    cluster-id: 99
    cluster-type: cluster_flow
    defrag: yes

  - interface: eth1
    cluster-id: 98
    cluster-type: cluster_flow
    defrag: yes

  - interface: default
```

Suricata escucha en modo promiscuo en ambas interfaces capturando todo el tráfico visible.

---

## Reglas

Las reglas se almacenan en `/var/lib/suricata/rules/suricata.rules` y se enlazan a `/etc/suricata/rules/`:

```bash
ln -s /var/lib/suricata/rules/suricata.rules /etc/suricata/rules/suricata.rules
```

Para actualizar las reglas manualmente:

```bash
suricata-update
systemctl restart suricata
```

## Logs — EVE JSON

Suricata genera logs en formato JSON en `/var/log/suricata/eve.json`.

### Tipos de eventos registrados

| Tipo | Descripción |
|------|-------------|
| alert | Alertas de reglas disparadas |
| dns | Consultas y respuestas DNS |
| http | Tráfico HTTP |
| tls | Handshakes TLS/SSL |
| ssh | Sesiones SSH |
| flow | Flujos de red |
| stats | Estadísticas del sistema |

### Ver alertas en tiempo real

```bash
tail -f /var/log/suricata/eve.json | grep '"event_type":"alert"'
```

### Ver todos los eventos

```bash
tail -f /var/log/suricata/eve.json
```

## Node Exporter

Instalado para monitorización desde Prometheus/Grafana.

```bash
wget https://github.com/prometheus/node_exporter/releases/download/v1.7.0/node_exporter-1.7.0.linux-amd64.tar.gz -O /tmp/node_exporter.tar.gz
tar xvf /tmp/node_exporter.tar.gz -C /opt/
mv /opt/node_exporter-1.7.0.linux-amd64 /opt/node_exporter
```

Servicio systemd:

```bash
cat > /etc/systemd/system/node_exporter.service << 'EOF'
[Unit]
Description=Node Exporter
After=network.target

[Service]
Type=simple
ExecStart=/opt/node_exporter/node_exporter
Restart=always

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable node_exporter
systemctl start node_exporter
```

Puerto: `10.1.1.36:9100`
