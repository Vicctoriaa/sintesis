# 🛡️ Suricata + jq + Wazuh Integration Guide

Guía completa para instalar **Suricata**, analizar logs con **jq** e integrarlo con **Wazuh** como SIEM.

---

# 📌 1. Instalación de Suricata

## 🔄 Actualizar sistema
```bash
apt update && apt upgrade -y
📦 Instalar Suricata
apt install suricata -y
🔍 Verificar instalación
suricata -V
🌐 Configurar interfaz de red

Ver interfaces disponibles:

ip a

Editar configuración:

nano /etc/suricata/suricata.yaml

Buscar sección:

af-packet:
  - interface: eth0

👉 Cambia eth0 por tu interfaz real (ej: ens33, ens18, etc.)

▶️ Iniciar Suricata
systemctl enable suricata
systemctl start suricata
📊 Ver logs en tiempo real
tail -f /var/log/suricata/eve.json
🧰 2. Instalación de jq (análisis de logs)
📦 Instalar jq
apt install jq -y
✔️ Verificar instalación
jq --version
🔍 Uso básico con Suricata
Ver alertas
jq 'select(.event_type=="alert")' /var/log/suricata/eve.json
Ver tráfico HTTP
jq 'select(.event_type=="http")' /var/log/suricata/eve.json
Tiempo real (alertas en vivo)
tail -f /var/log/suricata/eve.json | jq 'select(.event_type=="alert")'
🧩 3. Integración con Wazuh
🧠 Objetivo
Suricata genera logs en eve.json
Wazuh los lee
Wazuh los muestra en dashboard SIEM
📄 Configurar Wazuh Agent

Editar configuración:

nano /var/ossec/etc/ossec.conf

Añadir:

<localfile>
  <log_format>json</log_format>
  <location>/var/log/suricata/eve.json</location>
</localfile>
🔄 Reiniciar agente
systemctl restart wazuh-agent
