# Suricata — Configuración CT 106

> Este contenedor aloja Suricata, el motor de detección de intrusiones (IDS) del laboratorio. Suricata analiza el tráfico de red en tiempo real aplicando reglas para detectar amenazas, y escribe los eventos en formato JSON (`eve.json`). Ese fichero es consumido por el agente Wazuh (que lo envía al SIEM) y por el `eve-watcher.py` (que dispara playbooks Ansible de respuesta automática).

## Datos del contenedor

> Contenedor privilegiado con más recursos que el resto (2 cores, 1 GB RAM, 8 GB disco) porque Suricata es computacionalmente exigente: necesita inspeccionar cada paquete de red en tiempo real. El modo privilegiado es necesario para acceder directamente a las interfaces de red del host.

| Campo | Valor |
|-------|-------|
| CT ID | 106 |
| Hostname | suricata-ids |
| Imagen | debian-12-standard_12.12-1_amd64.tar.zst |
| Memoria | 1024 MB |
| Disco | 8 GB (local-lvm) |
| Cores | 2 |
| Privilegiado | Sí |

## 1. Instalación de Suricata

### Actualizar sistema

```bash
apt update && apt upgrade -y
```

### Instalar Suricata

```bash
apt install suricata -y
```

### Verificar instalación

```bash
suricata -V
```

### Configurar interfaz de red

```bash
ip a
nano /etc/suricata/suricata.yaml
```

Buscar sección:

```yaml
af-packet:
  - interface: eth0
```

> Cambia `eth0` por tu interfaz real

### Iniciar Suricata

```bash
systemctl enable suricata
systemctl start suricata
```

### Ver logs en tiempo real

```bash
tail -f /var/log/suricata/eve.json
```

## 2. Instalación de jq (análisis de logs)

```bash
apt install jq -y
jq --version
```

### Uso básico con Suricata

```bash
# Ver alertas
jq 'select(.event_type=="alert")' /var/log/suricata/eve.json

# Ver tráfico HTTP
jq 'select(.event_type=="http")' /var/log/suricata/eve.json

# Alertas en tiempo real
tail -f /var/log/suricata/eve.json | jq 'select(.event_type=="alert")'

# Alertas de severidad 1 en tiempo real
tail -f /var/log/suricata/eve.json | jq 'select(.event_type=="alert" and .alert.severity==1)'
```

## 3. Integración con Wazuh

### Objetivo

```
Suricata genera logs en eve.json
       ↓
Wazuh Agent los lee y envía al manager
       ↓
Wazuh Manager los muestra en dashboard SIEM
```

### Configurar Wazuh Agent

Editar `/var/ossec/etc/ossec.conf` y añadir:

```xml
<localfile>
  <log_format>json</log_format>
  <location>/var/log/suricata/eve.json</location>
</localfile>
```

```bash
systemctl restart wazuh-agent
```

## 4. Automatización — eve-watcher.py

### Objetivo

El `eve-watcher.py` monitoriza `eve.json` en tiempo real y, cuando detecta alertas de severidad alta, conecta via SSH a CT103 para ejecutar playbooks Ansible de respuesta automática.

```
Suricata genera alerta en eve.json
       ↓
eve-watcher.py detecta el evento (tail -f)
       ↓
SSH a CT103 (10.1.1.34:2222)
       ↓
ansible-playbook block-ip.yml / isolate-host.yml
```

### Instalación

```bash
mkdir -p /opt/soc
cp eve-watcher.py /opt/soc/eve-watcher.py
chmod 700 /opt/soc/eve-watcher.py

cp eve-watcher.service /etc/systemd/system/eve-watcher.service

touch /var/log/soc-eve-watcher.log
chmod 640 /var/log/soc-eve-watcher.log

systemctl daemon-reload
systemctl enable eve-watcher
systemctl start eve-watcher
```

### Configuración SSH — CT106 → CT103

El eve-watcher conecta a CT103 via SSH para lanzar Ansible. La clave pública de CT106 debe estar en el `authorized_keys` de CT103.

**Clave pública de CT106** (`suricata@soc.local`):
```
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIB90xyJnuALvzBDyqsGxV3RMdbJd3Wfm3urmBMRcGyWe suricata@soc.local
```

**Añadir en CT103:**
```bash
echo 'ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIB90xyJnuALvzBDyqsGxV3RMdbJd3Wfm3urmBMRcGyWe suricata@soc.local' \
  >> /root/.ssh/authorized_keys
```

### Logica de clasificacion de alertas

| Condicion | Accion |
|-----------|--------|
| sev=1, src IP externa | `block-ip.yml -e malicious_ip=<src_ip>` |
| sev=1, src IP interna | `isolate-host.yml -e target_vlan=<vlan_origen>` |
| sev=2, firma C2/malware, src externa | `block-ip.yml -e malicious_ip=<src_ip>` |
| sev=2, firma scan/nmap, src interna | `isolate-host.yml -e target_vlan=<vlan_origen>` |
| firma exploit/shellcode cualquier sev | `block-ip.yml -e malicious_ip=<src_ip>` |

**Cooldown:** 5 minutos por IP para evitar ejecuciones repetidas ante floods de alertas.

**Mapeo VLAN por subred** (para aislamiento de hosts internos):

| Subred | VLAN OpenWRT |
|--------|-------------|
| 10.1.1.0/27 | vlan10 (DMZ) |
| 10.1.1.32/27 | vlan20 (Servicios) |
| 10.1.1.64/27 | vlan30 (SOC) |
| 10.1.1.96/27 | vlan40 (Produccion) |
| 10.1.1.128/27 | vlan50 (Honeypot) |

### Gestion del servicio

```bash
# Estado
systemctl status eve-watcher

# Logs en tiempo real
tail -f /var/log/soc-eve-watcher.log

# Reiniciar
systemctl restart eve-watcher

# Detener
systemctl stop eve-watcher
```

### Prueba manual

Inyectar una alerta de prueba en eve.json:

```bash
echo '{"timestamp":"2026-04-24T14:20:00.000000+0000","event_type":"alert","src_ip":"5.5.5.5","dest_ip":"10.1.1.36","proto":"TCP","alert":{"action":"allowed","gid":1,"signature_id":2404000,"rev":1,"signature":"ET MALWARE Test C2 Traffic","category":"A Network Trojan was Detected","severity":1}}' \
  >> /var/log/suricata/eve.json
```

Verificar en el log del watcher:
```bash
grep -E "block-ip|5\.5\.5\.5|OK|FALLO" /var/log/soc-eve-watcher.log
```

Verificar en CT103:
```bash
grep "5.5.5.5" /var/log/soc-incidents.log
cat /etc/suricata/rules/local.rules
```

## 5. Reglas locales de Suricata

**Fichero:** `/etc/suricata/rules/local.rules`

Las IPs bloqueadas por el SOC se añaden aqui automaticamente via `block-ip.yml`:

```
drop ip <ip_maliciosa> any -> $HOME_NET any (msg:"SOC Block - Malicious IP <ip>"; sid:9000001; rev:1;)
```

Para recargar reglas en caliente sin reiniciar Suricata:

```bash
suricatasc -c reload-rules
```

Para ver las reglas activas:

```bash
cat /etc/suricata/rules/local.rules
```

## Comandos utiles

```bash

# Estadisticas de alertas por firma
jq 'select(.event_type=="alert") | .alert.signature' /var/log/suricata/eve.json | sort | uniq -c | sort -rn | head -20

# Reglas de bloqueo activas
cat /etc/suricata/rules/local.rules

# Recargar reglas
suricatasc -c reload-rules

# Validar configuracion
suricata -T -c /etc/suricata/suricata.yaml
```
