## 3. Configuración principal

### `named.conf.options`

```
options {
    directory "/var/cache/bind";
    recursion yes;
    allow-recursion { 10.1.1.0/24; localhost; };
    allow-query { 10.1.1.0/24; localhost; };
    forwarders {
        8.8.8.8;
        1.1.1.1;
    };
    dnssec-validation auto;
    listen-on { 10.1.1.34; localhost; };
};

statistics-channels {
    inet 127.0.0.1 port 8080 allow { 127.0.0.1; };
};
```

| Parámetro | Valor | Descripción |
|-----------|-------|-------------|
| `recursion` | yes | Permite resolución recursiva |
| `allow-recursion` | `10.1.1.0/24` | Solo la red interna puede hacer recursión |
| `allow-query` | `10.1.1.0/24` | Solo la red interna puede consultar |
| `forwarders` | `8.8.8.8`, `1.1.1.1` | DNS externos para dominios no locales |
| `dnssec-validation` | auto | Validación DNSSEC activada |
| `listen-on` | `10.1.1.34`, `localhost` | Solo escucha en la IP interna |

### `named.conf.local`

## 6. Comandos de gestión

### Recargar zonas sin reiniciar
```bash
rndc reload
```

### Verificar sintaxis antes de recargar
```bash
named-checkconf
named-checkzone soc.local /etc/bind/zones/db.soc.local
named-checkzone 1.1.10.in-addr.arpa /etc/bind/zones/db.10.1.1
```

### Consultas de prueba
```bash
# Resolución directa
dig @10.1.1.34 grafana.soc.local +short
dig @10.1.1.34 soar.soc.local +short

# Resolución inversa
dig @10.1.1.34 -x 10.1.1.66 +short
dig @10.1.1.34 -x 10.1.1.37 +short

# Comprobar forwarder externo
dig @10.1.1.34 google.com +short
```

### Reiniciar servicio
```bash
systemctl restart named
systemctl status named
```

## 7. Procedimiento para añadir un nuevo host

1. Añadir registro A en `/etc/bind/zones/db.soc.local`
2. Añadir registro PTR en `/etc/bind/zones/db.10.1.1`
3. Incrementar el serial en ambos ficheros (formato `YYYYMMDDnn`)
4. Verificar sintaxis: `named-checkzone soc.local /etc/bind/zones/db.soc.local`
5. Recargar: `rndc reload`
6. Verificar: `dig @10.1.1.34 <nuevo-host>.soc.local +short`
